# Plan 0: Integration Contracts — Implementation Plan

**Last Updated:** 2026-03-02

---

## Executive Summary

Define all shared types, DB schema changes, API route contracts, SSE event formats, and a backfill migration script for VIE Bible v1.4. This plan is the **gate** — Plans 1-4 (Frontend, API, Summarizer, Explainer+Admin) cannot start until these contracts are merged.

**Scope:** Types only, no route implementations. The backfill script creates indexes and sets defaults on existing documents.

**Effort:** ~2-3 days total, all tasks sequential within sections but sections 1-3 can be done in parallel.

---

## Current State Analysis

### Shared Types (`packages/types/src/index.ts` — 948 lines)
- **VideoCategory** exists as a union of 11 values (`cooking`, `coding`, `travel`, etc.)
- **VideoSummary** has `tldr`, `keyTakeaways`, `chapters`, `concepts`, `masterSummary`, `rawTranscriptRef`, `generation`
- **VideoResponse** has standard fields + progressive summarization fields + `context?: VideoContext`
- **SSE events** are typed as `SSEStreamEvent` union (11 event types) — no `detection_result` event yet
- **No OutputType, UserTier, ShareInfo, TierLimits, or PublicVideoSummary** types exist

### DB Indexes (`api/src/plugins/mongodb.ts`)
- `videoSummaryCache`: indexes on `{youtubeId, isLatest}`, `{youtubeId, version}`, `{status}`
- `users`: unique index on `{email}`
- No `outputType`, `shareSlug`, or `expiresAt` indexes exist

### Migration Script Patterns
- **JS pattern** (`migrate-indexes-v2.js`): Direct MongoDB driver, `updateMany`, `createIndex`, idempotent
- **TS pattern** (`reset-database.ts`): Uses `tsx` runner, interactive confirmation, reports counts
- **Python pattern** (`backfill-transcripts.py`): Dry-run support, batch processing, stats reporting

### DI Container (`api/src/container.ts`)
- Manual factory pattern: `createContainer(db, logger)` returns typed `Container`
- Repositories + Services + Clients, all wired manually

### LLM Usage Tracking
- `llm_usage` collection with `UsageRecord` model in `packages/llm-common`
- Already tracks `tokens_in`, `tokens_out`, `cost_usd`, `video_id`, etc.
- `totalTokens` for videoSummaryCache would aggregate from here

## Risk Assessment

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| Type changes break existing consumers | High | Low | All new fields are optional — fully backward compatible |
| Backfill script corrupts data | High | Low | Dry-run mode, only `$set` with `$exists: false` guards |
| TTL index deletes data prematurely | High | Low | `expiresAt` defaults to `null` (never expires), only set explicitly |
| Shared package version drift | Medium | Medium | Single `packages/types` import across all TS consumers |
| Python services can't use TS types | Low | N/A | Python has own Pydantic models — mapping constant is documentation |
