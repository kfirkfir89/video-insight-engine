## Key Decisions

### 1. OutputType as String Union (not Enum)
**Decision:** Use TypeScript string union `type OutputType = 'recipe' | ...` not an enum.
**Why:** Matches existing `VideoCategory` pattern. String unions are more ergonomic in TS, JSON-serializable without transform, and work across Python boundary (Python uses its own Pydantic models).

### 2. All New Fields Optional
**Decision:** `outputType`, `shareInfo` on existing types are optional (`?`).
**Why:** Backward compatibility. Existing documents don't have these fields. Frontend/API gracefully handles `undefined`. Backfill sets defaults but code doesn't assume they exist.

### 3. Backfill as TypeScript (not JS or Python)
**Decision:** `scripts/backfill-v1.4.ts` using `tsx` runner.
**Why:** Matches `reset-database.ts` pattern. Can import from `packages/types` for type safety. Uses MongoDB Node driver directly (same as API).

### 4. TTL Index with null Default
**Decision:** `expiresAt` defaults to `null`, TTL index only fires on non-null Date values.
**Why:** MongoDB TTL indexes ignore documents where the field is `null` or missing. This means existing content never expires. Only new content created with tier rules gets an expiration.

### 5. CATEGORY_TO_OUTPUT_TYPE in Shared Types
**Decision:** Put the mapping constant in `packages/types/src/index.ts`.
**Why:** Both API (TypeScript) and frontend (TypeScript) need it. Summarizer (Python) has its own copy in `services/summarizer/src/services/output_type.py` (Plan 3).

### 6. Sparse Unique Index for shareSlug
**Decision:** `{ shareSlug: 1 }` with `unique: true, sparse: true`.
**Why:** Most documents won't have a `shareSlug`. Sparse index excludes null/missing values from uniqueness check, so millions of unshared documents don't conflict.

## API Route Contracts (for Plan 2 reference)

### Share Routes
```
POST /api/share/:videoSummaryId  →  { shareSlug: string, shareUrl: string }
GET  /api/share/:slug            →  PublicVideoSummary
POST /api/share/:slug/like       →  { likesCount: number }
```

### Override Route
```
PATCH /api/videos/:id/override-category
  Body: { category: VideoCategory }
  →  { outputType: OutputType }
```

### Payment Routes
```
POST /api/payments/webhook       →  (Paddle signature verified) → 200
GET  /api/payments/checkout      →  { checkoutUrl: string }
GET  /api/tier                   →  { tier: UserTier, limits: TierLimits }
```

### SSR Route
```
GET /s/:slug  →  HTML (OG tags + JSON-LD + PublicVideoSummary embedded)
```
