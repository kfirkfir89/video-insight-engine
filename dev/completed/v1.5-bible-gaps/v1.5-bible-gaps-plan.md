## Phase 1: Type Safety & Contracts

### 1.1 Add User Type (S)
- Define `User` interface in `packages/types/src/index.ts`
- Fields: `_id`, `email`, `name`, `username: string | null`, `displayName: string | null`, `tier`, `themePreference`, `referralSlug`, `paddleCustomerId`, `paddleSubscriptionId`, timestamps
- `username` is unique sparse index (can be null, but unique when set)
- Mark `username`, `displayName`, `themePreference`, `referralSlug` as optional (migration safety)

### 1.2 Add ThemePreference Type (S)
- `type ThemePreference = 'system' | 'light' | 'dark'`
- Add to User type: `themePreference?: ThemePreference`

### 1.3 Add Like Collection Type (S)
- `interface Like { _id: ObjectId, outputId: ObjectId, userId: ObjectId | null, ipHash: string, createdAt: Date }`
- Compound index: `{ outputId, ipHash }` unique

### 1.4 Document OutputType Naming Decision (S)
- Add comment block in `packages/types/src/index.ts` above `CATEGORY_TO_OUTPUT_TYPE`
- State: "This is the SINGLE source of truth for category→outputType mapping. All services import from here."
- Python services mirror this mapping — document the mirror locations

### 1.5 Ensure Migration Safety (S)
- Verify all new fields have `?` (optional) in TypeScript
- Add JSDoc: `@migration Mark required after backfill-v1.4.ts confirmed`

### 1.6 Create Rollback Script (M)
- `scripts/rollback-v1.4.ts` — removes TTL index, sparse unique on shareSlug
- Idempotent (safe to run multiple times)
- Does NOT delete data, only indexes

## Phase 2: Frontend Design Corrections

### 2.1 Fix Glass System (M)
- `.glass` → solid `var(--vie-card)` bg, glass border/radius, NO blur
- `.glass-elevated` → real `backdrop-filter: blur(20px)` for nav/hero/active only
- Add `@supports (backdrop-filter: blur(1px))` fallback
- Update existing components using `.glass` (verify each usage)

### 2.2 Create GlassElevated Component (S)
- `apps/web/src/components/ui/glass-elevated.tsx`
- Props: `className`, `children`, `as` (polymorphic)
- Only for: sticky nav, hero cards, active card, modal overlays
- Max 3-4 per screen (document in component JSDoc)

### 2.3 Restructure Accent Palette (S)
- In `index.css`, add comment block:
  - Primary (use freely): `--vie-coral`, `--vie-plum`, `--vie-mint`
  - Rare (semantic only): `--vie-sky`, `--vie-honey`, `--vie-rose`, `--vie-forest`, `--vie-peach`
  - Rule: max 4 accent colors on any single screen
- Audit existing usage for compliance

### 2.4 Anonymous Output Tracking (M)
- Add to `auth-store.ts`: `anonymousOutputCount` persisted to localStorage
- Trigger auth prompt after 3rd output or on save/share
- Decision: **localStorage-only** for anonymous outputs (simplest, no DB schema change)
- On auth: migrate localStorage cache keys to user account

### 2.5 Fix Skeleton Loading (S)
- Skeleton cards use `var(--vie-card)` solid bg with shimmer
- No glass/blur on empty content

### 2.6 SwipeableView Decision (S)
- Recommend: `react-swipeable` (lightweight, React 19 compatible)
- Verify peer deps before adding
- Alternative: defer to Phase 2 (desktop-first MVP)

## Phase 3: API Missing Routes & Hardening

### 3.1 Add PATCH /api/users/me/preferences (M)
- Accept: `{ theme?: ThemePreference, displayName?: string, username?: string }`
- Validate username uniqueness (sparse index handles nulls)
- Return updated user
- Auth required

### 3.2 Add PATCH /api/videos/:id/blocks (M)
- Accept: `{ blocks: Array<{ blockId: string, data: Record<string, any> }> }`
- Validate ownership
- Merge edited blocks into videoSummaryCache document
- Return success with updated blocks

### 3.3 Dedup View Counts (M)
- `GET /api/share/:slug` — check `views` collection for `{ slug, ipHash, viewedAt }`
- Only increment if no view from same IP in last 24h
- Or: use fire-and-forget with TTL collection (simpler)

### 3.4 Decide Production SSR Architecture (S — decision only)
- **Recommended: Vercel rewrites `/s/*` to Railway API**
- Simplest deployment, Railway already handles HTML rendering
- Document in `docs/INFRASTRUCTURE.md`
- Config: `vercel.json` rewrite rule

### 3.5 Rate Limit Likes per IP per Slug (S)
- Current: 10/min global rate limit on likes
- Enhance: max 1 like per IP per slug per hour (compound check in service)

## Phase 4: Expiration & Business Logic

### 4.1 Handle Pro → Free Downgrade (M)
- **Decision: Grandfather existing, new outputs get 30d TTL**
- On downgrade: set `expiresAt = now + 30d` on all outputs created AFTER downgrade
- Previously-created outputs keep no expiration
- Shared outputs never expire (growth driver)

### 4.2 Handle Shared Output Expiration (S)
- **Decision: Shared outputs never expire**
- Add guard: if `shareSlug` exists, skip TTL application
- SEO and viral loops depend on permanent shared links

### 4.3 Defer Day-25 Warning to Phase 2 (S)
- No SES setup in v1.5
- Add TODO in `payment.service.ts` with Phase 2 tag
- For now: show in-app banner when output is 5 days from expiry

### 4.4 Override Timing Guard (S)
- **Decision: Only allow override BEFORE first chapter_ready SSE event**
- In `override.py`: check if pipeline has emitted any `chapter_ready`
- If mid-stream override: return 409 Conflict with message

## Risk Assessment

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| Glass refactor breaks existing components | High | Medium | Audit all `.glass` usages before changing |
| Username uniqueness conflicts at scale | Low | Low | Sparse index + validation + retry |
| Anonymous→auth migration loses data | High | Medium | localStorage approach = client-side only, low risk |
| View dedup adds latency to GET | Medium | Low | Fire-and-forget pattern, TTL collection |
| SSR Vercel→Railway rewrite adds latency | Medium | Medium | Cache HTML at edge, 60s TTL |
| CI/CD pipeline adds 5-10min to merges | Low | High | Parallelize test matrix, cache dependencies |
